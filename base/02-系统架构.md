# 02 · 系统架构（通用，所有模板共享）

## 一、两层分离

**生成层（Node 脚本，可缓存可重跑）→ 渲染层（Remotion，纯数据驱动）。**

```
输入(文案 / 源视频 / 词表…)
  → 生成层：分镜/剪辑 + 翻译/拼音 + TTS 配音 + (按需)出图/剪辑 + 尺寸/时间轴归一化
  → manifest.json（渲染层唯一输入）
  → 渲染层：一个 layout 组件读 manifest → remotion render → 成片.mp4
```

- 生成层每步产出**可缓存的中间文件**（按输入 hash），改一处不重烧整片（见 [[03-流水线与manifest]]）。
- 渲染层**不含业务内容**：它只按 manifest 摆放画面/字幕/音频/特效。加模板通常不动渲染层核心，只加一个 layout。

## 二、模板系统（本项目的核心抽象）

一个"内容形态"= 一个**模板**，位于 `templates/<模板>/`：

```
templates/<模板>/
├── template.json     该模板的运行期配置(深合并进 config/settings.json)：meta/字幕/字体/音色映射/画风开关…
├── authoring.md      该模板的唯一作者指南(Claude 按它把输入变成 script/clips，再触发 build)
├── build.mjs         该模板的生成流水线，导出 async build(ctx) → 返回 manifest 对象
└── (按需) characters/ prompts/ motion.json branding/ …   该模板私有资产
```

**编排器 `scripts/build.mjs`（通用，不含模板逻辑）：**
`node scripts/build.mjs <videoId>` → 从 `catalog.json` 查该视频的 `template` → 加载 `loadSettings(template)`（全局 + 模板深合并）→ 动态 import `templates/<template>/build.mjs` 的 `build(ctx)` → 落盘 `manifest.json` → 回写 catalog 状态。

`build(ctx)` 收到的 `ctx = { videoId, dir, ROOT, settings, rel, ensure }`，必须返回 `{ meta, beats }` 形态的 manifest 对象。

## 三、配置合并：全局默认 + 模板覆盖

- `config/settings.json`：**只放跨模板通用**的运行期默认——语言清单、TTS 引擎、旁白语言。**不放任何模板私有参数**。
- `templates/<模板>/template.json`：该模板私有的一切运行期配置，加载时**深合并覆盖**全局。
- 改一个模板的字号/配色/音色/画幅 → 只改它的 `template.json`，不动别的模板、不动代码。

## 四、总账与分片

- **`catalog.json`**：视频总账，一条视频一条记录（`id / title / template / shard / lang / status / manifest / beats`）。扛上千条时的索引与状态。
- **shard = 年/月/日（`YYYY/MM/DD`）**：产物路径 `public/videos/<shard>/<id>/`。按天归档，避免单目录几千项。代码只把 `shard` 当路径片段拼接，改层级无需动脚本。

## 五、单条视频的目录产物规范

每条视频**一个自包含目录**，可独立重渲染/删除，链路可追溯：

```
public/videos/<shard>/<id>/
├── input.md / clips.json / script.json …   ① 输入 & ② 我(Claude)设计的脚本(因模板而异)
├── audio/  images/  work/ …                 ③ 生成层中间产物(配音/出图/剪辑片段/ASR…)
├── manifest.json                            ④ 渲染数据(build 汇总，渲染层读它)
└── 成片.mp4                                  ⑤ 最终成片(gitignore，只本地生成，见 [[04-成本与铁律]])
```

- **原则**：人会编辑的（提示词/角色/参数）进 `config/` 或 `templates/`；机器生成的产物进 `public/videos/`。
- 一个目录看下去，就知道这条片子怎么从输入长成成片。②"脚本"层是**我设计**的东西（不是机械切句）：怎么切拍、怎么共图/共段、每段怎么配音、每张图/每段素材怎么定——全在这里定死，后续配音/出图/剪辑只照着执行。具体字段见各模板 authoring。

## 六、渲染层与 layout 注册

- 渲染层入口 `src/Video.tsx` 读 `manifest.meta.layout` → 从 `src/layouts/registry.ts` 取对应 `LayoutModule` 渲染。
- 加模板若版式不同：写 `src/layouts/<id>.tsx` 导出 `LAYOUT`（见 `src/layouts/types.ts`），在 registry 注册即可，`Video.tsx` 不用改。
- 未知/缺省 layout 回落到兜底版式，不崩。
