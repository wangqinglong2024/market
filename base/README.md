# base · 项目基座（通用、跨模板、必须遵循）

> 这里是**基座**：与任何一个具体视频模板都无关、但**所有模板都必须遵循**的东西。
> 之前的 `plan/` 是围绕单个模板（guoxue-jinju）写的，混入了大量该模板的私有参数；
> 现已整合重构为通用基座——**只留跨模板通用的规律、架构、铁律**，模板私有参数一律下沉到各自
> `templates/<模板>/`（`template.json` + `authoring.md`）。

## 这套项目是什么

一台**「文案 / 素材 → 竖屏短视频成片」的自动化流水线**，面向东南亚（首发越南）TikTok。
不同的内容形态（国学金句、中文学习字幕、正念翻牌…）各是一个**模板**；基座提供它们共享的
底层规律、目录约定、构建编排器、渲染层与铁律。

## base 与 templates 的分工（一句话记住）

| 层 | 放什么 | 例子 |
|---|---|---|
| **base/**（本目录） | 跨模板**通用**的规律 / 架构 / 铁律 | 底层规律、两层架构、缓存、fal 铁律、mp4 gitignore |
| **config/settings.json** | 跨模板**通用**的运行期默认 | 语言清单、TTS 引擎、旁白语言 |
| **templates/&lt;模板&gt;/** | 该模板**私有**的一切 | 内容结构、画风、角色、字号、音色映射、build.mjs |

> 判断某条规则该写哪：**问「换一个模板它还成立吗」**。成立 → base；只服务某模板 → 该模板目录。
> **非通用参数不要写进 base**（本次重构已把它们从 plan 里删干净）。

## 加一个新模板（标准动作）

1. 建 `templates/<新模板>/`：`template.json`（深合并进 `config/settings.json`）+ `authoring.md`（该模板作者指南）+ `build.mjs`（导出 `async build(ctx)`，见 [[02-系统架构]]）。
2. 若字幕/版式不同：写 `src/layouts/<layout>.tsx` 导出 `LAYOUT`，在 `src/layouts/registry.ts` 注册（`Video.tsx` 不用改）。
3. `catalog.json` 加条目并写 `"template": "<新模板>"`。
4. 该模板**私有**的规律写进它的 `authoring.md`；只有确认是**跨模板通用**的，才回写到 base。
