# 03 · 通用流水线、manifest 与共享机制

> 本文只写**跨模板通用**的流水线概念与渲染机制。具体某模板"分几步、每步做什么"由它的 `build.mjs`
> 和 `authoring.md` 决定；但下面这些**机制是共享的**，各模板复用同一套。

## 一、生成层通用步骤（能力菜单，模板按需取用）

| 能力 | 通用脚本 | 说明 |
|---|---|---|
| 分镜 / 剪辑设计 | （会话内 Claude 产出 `script.json` / `clips.json`） | 把输入按 [[01-创作准则]] 设计成时间线，**这是最关键的一步** |
| 语音识别 ASR | `scripts/asr.mjs` | 火山「录音文件识别大模型·极速版」，出逐词时间戳（字幕/剪辑对齐用） |
| 配音 TTS | `scripts/tts.mjs` | 火山 TTS，出 `.mp3` + 真实时长(ms) + 可选字级时间戳(`with_timestamp:1`) |
| 出图 | `scripts/gen-image.mjs` | fal.ai，按角色数路由模型（见 [[04-成本与铁律]]）；仅"出图类"模板用 |
| 时长探测 / 剪辑 | `scripts/lib/media.mjs` | 跨平台 `ffprobe` 探时长；Remotion 内置 ffmpeg 剪拼 |
| BGM / 音效 | `scripts/gen-bgm.py` / `gen-sfx.py` | **本地合成**，禁 API 生成音乐（见 [[04-成本与铁律]]） |

**时长是唯一节奏源**：每拍时长 = 该拍配音/素材真实时长（+尾部留白）。配音变快 → 该拍变短 → 画面/运镜节奏自动跟着变，不用另调。

## 二、manifest.json 通用形态

渲染层的唯一输入。通用骨架（具体字段因 layout 而异）：

```jsonc
{
  "meta": {
    "fps": 30, "width": 1080, "height": 1440,   // 画幅通用竖屏 3:4；模板可覆盖
    "layout": "<layout id>",                     // → src/layouts/registry.ts
    "captions": { /* 三行配色/字号/间距，模板私有，见其 template.json */ },
    "fonts":    { /* 字体栈，模板私有 */ }
    // + 该 layout 需要的其它 meta(source/subtitle/bgm…)
  },
  "beats": [
    { "id": "...", "startMs": 0, "durationMs": 2200, /* + 该 layout 的字幕/画面/音频/特效字段 */ }
  ]
}
```

- **beat（一拍）= 时间线最小单位**：它绑定"这段时间里 画面 / 配音 / 字幕 / 特效"，天生同步。
- 字幕/画面/配音都从**同一个 beat** 的时间窗派生，共享 `[startMs, startMs+durationMs]`。

## 三、按输入 hash 缓存（成本与速度命脉）

- 每步按**输入 hash** 缓存：分镜、TTS(文本+音色)、出图(prompt+seed+参考图)、ASR(音频)…
- 命中缓存不重生成；改一句只重算受影响的拍，不重烧整片。
- 出图/合成的 **seed 固定**，可复现、可精确重抽。
- ⚠️ 缓存多按**输出路径/内容 hash** 命中——改语速/改参数但输出路径没变时，要**手动删掉受影响的中间产物**再重跑。

## 四、字幕与卡拉OK（共享机制）

- **一屏一句短句**：字幕按短句切细，一次只显示当前正在读/播的一句，随音频逐句切换。**文字是视觉重点**，短句才放得大。
- **多语三行**：拼音 / 中文 / 当地语言（首发越南语），**每层单行、不换行**；单行放不下时由 `FitLine` 量宽整行等比缩小兜底（但切句先保证本来就放得下）。
- **逐词/逐字卡拉OK**：读到哪个词/字，哪个**点亮高亮**（可放大跳一下）。时间源：
  - 配音是我们自己 TTS 的 → 用火山**字级时间戳**（`with_timestamp:1` → `addition.frontend.words[]`，毫秒，自然停顿自带）。标点挂在前一字上，入 manifest 前要剥掉只留汉字，与中文逐字对齐。
  - 素材是原视频原声 → 用 **ASR 逐词时间戳**对齐。
  - 当地语言语序≠中文、无法逐字锁死 → **按本卡时长均匀铺满**，视觉上与中文一起往前跑。
- **翻译**：文言/成语/口语一律**意译**、地道口语，非逐字直译（见 [[01-创作准则]] 六）。
- **多音字引导**：字幕对≠读音对，TTS 也会读错。遇多音字给该拍加只供合成、不上屏的引导字段（如把"难nán"写成"男"）；⚠️ **引导文本的汉字数必须与上屏中文完全一致**，否则逐字跳字错位。出片后逐句听一遍多音字。

## 五、运镜（共享机制）

- `motion.json`（模板私有预设库）支持组合字段：`scale`(推拉)+`panX/panY`(平移)+`driftX/driftY`(正弦漂移，静图也"呼吸")+`rotate`(轻旋)+`ease`。
- 常用预设：`push-in / pull-back / pan-left / pan-right / climb-up / ken-burns / sway / pop / still`。
- **多样化铁律**：别全片一个运镜；按每拍内容选，相邻两拍尽量不同预设。运镜是"没特效也好看"的主力。
- beat 间用 `@remotion/transitions` 做转场（淡入/滑入/翻页），只在**场景/片段切换处**转场；共图/共段的相邻拍不换图、不转场、运镜连续。

## 六、特效（共享机制，克制使用）

- **`src/fx/` 是开放、可扩展的 React 特效库，不是固定集合**：想要新效果就写一个 `useCurrentFrame()` 驱动的组件放进 `src/fx/`，在 `src/layouts/shared.tsx` 的 `EffectsLayer` 加一个 `case` 即接入，无数量上限。现有的一些：`emotion/ComicPops`、`emotion/EmojiRain`、`emotion/ScorePop`、`distortion/ZoomBlur`（可自由增删）。转场类（白闪/变焦冲击/上滑淡入）由各 layout 自带。
- **逐帧确定性（唯一硬约束）**：特效必须是 `useCurrentFrame()` 的函数、`random(seed)` 种子化、接 `durationInFrames`。🚫 禁用 CSS animation / requestAnimationFrame / 实时 Canvas 循环（Remotion 逐帧截图，否则闪、不可复现）。
- **克制是取向、不是硬规则**：内容为主、特效为辅——绝大多数拍靠画面+运镜+配音+字幕就够；特效落在情绪高点/正反馈/需要强调处，别抢字幕主视觉。用几个、用哪个由内容决定，各模板可自己定尺度。

## 七、封面（★ 全局规则：每条视频都要出一张封面）

**任何模板、任何一条交付的视频，除了成片，都要额外生成一张封面图**（缩略图）。封面 = **原视频的一帧 + 文字钩子**。

- **底图**：从原视频里挑**最抓人的一帧**（名场面/冲突/反转/高唤醒的画面），抽成 `cover-bg.jpg`。挑图和挑开场钩子同理（见 [[01-创作准则]] 一），**不是随手截一张**。
- **文字**：由 Claude **按这条视频的实际内容拟**，每条不同——一句**中文钩子大字** + 一句**当地语言(越南语)钩子**。要**抓人**（好奇缺口/利害/高唤醒），**不要瞎搞**、不要平铺直叙的说明。
- **版式与内容(视频)一致（★ 用户 2026-07-13，防裁切遮挡）**：`src/Cover.tsx` 的 `cover` Composition = 1080×1440，**上下与中间间隔均 120**：`120 / 图片区720 / 120 / 文字区360 / 120`。图整帧完整呈现(不裁主体)居中在图片区，文字整组垂直居中在文字区，同帧模糊铺底填满画布。**所有主体/文字都落在中间安全带**，平台裁切封面后不会被遮挡。
- **渲染**：每条视频写一个 `cover.json`（`image/title/subtitle/tag/颜色`），命令：
  ```bash
  # 1) 抽最抓人的一帧(ASCII 源，见 [[04-成本与铁律]] 五)
  node_modules/.bin/remotion ffmpeg -y -ss <秒> -i <ASCII源.mp4> -frames:v 1 -q:v 2 <dir>/cover-bg.jpg
  # 2) 写 <dir>/cover.json，再渲染封面（命名规则见 [[02-系统架构]] 五：<id>-cover.png）
  node_modules/.bin/remotion still src/index.ts cover <dir>/<id>-cover.png --props=<dir>/cover.json
  ```
- **产物**：`cover.json`(配置) + `cover-bg.jpg`(底图) + `<id>-cover.png`(成品)，都放该视频目录（见 [[02-系统架构]] 五）。PNG 不受 mp4 的 gitignore 限制。成片同理命名 `<id>.mp4`。
- 加新的封面风格/版式：改 `src/Cover.tsx`（它是通用组件，跨模板复用）。
