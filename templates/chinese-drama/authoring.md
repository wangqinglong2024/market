# `chinese-drama`：《毒嫁》创作与生产说明

当前唯一 canonical 系列是全中国人物的古装权谋穿越剧《毒嫁》。核心不是讲知识，而是用敌婚、复仇、取证和先婚后爱持续制造高唤醒剧情。所有角色说普通话，拼音＋中文＋越南文仅作为三行影视字幕。

## 唯一事实源

续写和构建只读取：

1. `story/series.json`
2. `story/state.json`
3. `story/00-bible.md`
4. `story/arc-01.md`
5. 当前集目录的 `episode.json`
6. 当前集目录的 `generation-plan.json`

任何旧角色、旧故事包、旧图、旧提示词或历史 manifest 都不能反向污染 canonical。

## 默认只做设计

在用户明确批准之前，只允许：

- 写故事、逐镜脚本、episode JSON、提示词、预算和本地校验；
- 计算内容哈希和预检报告；
- 查看现有文件与运行不产生媒体的测试。

禁止调用 fal.ai、TTS、图像、视频、声音克隆或其他媒体生成服务；禁止创建伪审批文件；禁止把占位素材宣称为成片。`build.mjs` 在设计模式必须停在任何付费导入之前。

## 生产阶段的双重人工闸门

只有用户明确要求生成后，才依次进行：

1. 锁定 episode、generation plan、template 与虚构角色 canonical 哈希；
2. 单独批准关键帧阶段；
3. 每张关键帧人工验脸并记录哈希；
4. 再单独批准 I2V 与声音阶段；
5. 任何付费资产最多尝试一次，失败后必须重新向用户说明并征得批准，不自动重试。

## 内容硬标准

- 单集 22—26 秒，目标 24 秒，30fps；每个 shot 必须落在整数帧。
- 0—1 秒同时出现至少两个触发器；E01 为婚姻、仇人、毒酒、生死四触发。
- 1—3 秒完成不可逆动作，并让观众能复述唯一主问题。
- 主问题答案不得早于全片 80%；答案之后必须再出现更危险的新事实。
- 每三秒至少一次动作、信息、立场或权力变化；禁止站桩解释。
- 现代 AI 林晚冷开 0.6—1.0 秒，后期普通话配音，最后动作与同脸古装沈知微 match cut。
- AI 真视频覆盖至少 80%；Remotion 推拉、闪光和静态图不算真实动态覆盖。
- AI 视频单条只规划 3—5 秒整数时长，`generateAudio=false`。

## 故事逻辑硬标准

- 人物行动只能使用其已经获得的事实；观众看到不等于角色知道。
- 林晚只擅长文书鉴定，不能凭专业判断毒物、伤势或未知历史。
- 反派必须因利益行动，不能为了送线索而随身携带自证罪名的物件。
- 爱情动作必须同时完成取证、遮挡、伪装、救命或阵营变化。
- 萧玦确实签过拘捕令，必须承担后果；不能用误会洗白。
- 季终翻案依赖物证、记录和不同证人互证，不能靠反派自曝。

## 三行影视字幕

每一句 utterance 必须同时写：

```json
{
  "speakerId": "shenzhiwei",
  "mode": "dialogue",
  "pinyin": "nǐ zǎo jiù zhī dào",
  "textZh": "你早就知道。",
  "subtitleVi": "Chàng biết từ trước.",
  "startOffsetMs": 0,
  "estimatedDurationMs": 1200
}
```

- 拼音必须使用字母声调，不用数字声调。
- 中文与后期普通话配音逐字一致。
- 越南文优先短、自然、有关系语气；称谓要随亲疏和权力变化。
- 三行由 Remotion 后期合成，AI 图像和视频提示词不得要求画文字、字幕、边框或版式留白。
- 一屏只显示当前一句；不能加入词卡、解释、跟读、高亮教学或学习 CTA。

## 关键帧成本路线

- 每集最多一张两人及以上关键帧使用 `fal-ai/nano-banana-pro/edit`。
- 其余零人或单人关键帧全部使用 `fal-ai/flux-pro/kontext`。
- 多人关系尽量用单人反打、手部物证、遮挡和同一动态片段切窗口解决。
- 每个资产 `maxAttempts=1`，无自动付费重试。

## 付费前固定顺序

`故事圣经 → 单集主问题 → 逐镜因果 → 三行字幕 → schema → 状态哈希 → generation plan → 成本上限 → 本地 preflight → 用户批准 → 关键帧 → 人工验脸 → I2V/TTS → Remotion`

本轮必须停在“用户批准”之前。
