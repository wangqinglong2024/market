# chinese-drama 出图铁律（真人质感·连续剧）

> 画风=**电影级真人质感 cinematic photorealistic**（旧版 webtoon 作废）。以下为强制约束，违反即废图。

## 1. 身高与身材（除老年人外，全员适用）
- **男性**：身高 **1.80m 以上**，身材高挑、肩宽、精瘦结实（lean athletic）。
- **女性**：身高 **1.70m 以上**，身材**非常苗条纤细、腿长**（tall slim slender long-legged）。
- **老年人豁免**：顾老夫人等长辈不受身高/身材/服装规则约束，按角色年龄自然呈现。
- 身高/身材是**不变量**，写进每个角色 `canonical.md`，每拍复用。

## 2. 女性服装（默认显身材 · 场景优先 · 连戏）
- **默认原则**：女性默认穿**美丽、时尚、显身材**的服装（短裤 / 短裙 / 吊带 / 露肩 / 收腰连衣裙，得体不低俗，SFW）——这是**总体倾向**，用于没有特定要求的日常/出街/居家场景。
- **★场景优先（铁律）**：当**当集场景明确要求某套服装时，以场景为准，覆盖默认原则**。例：婚礼当晚=红嫁衣、守灵/奔丧=素服白衣、当值女佣=制服、正式晚宴=礼服、雨中/睡前各按情境。别把"显身材短裙"硬套到这些场景上。
- **★连戏（铁律）**：**同一场景 / 同一时间线内，服装必须前后一致**（如婚礼当晚从头到尾都是同一件红嫁衣，不许一拍嫁衣、下一拍换短裙）。"每集换装"指**集与集之间、或换了场景/时间**才换，不是一集内乱换。
- 落地：服装由**每拍 `shot.content`** 现写覆盖，`canonical.md` 只锁脸/发/身材、不锁死某套衣服；同一场景多拍要在 shot 里写"同前一套 XX 服装"保连戏。

## 3. 出图输入=参考人物图 + 当集服装 + 当集场景（三件套）
- 每次真的出图，除了喂**该角色定妆参考图**（锁脸），**必须**在 prompt 里同时写清：
  1. **当集这一套服装**（第2条，具体到款式/颜色）；
  2. **当集这一个场景**（地点/光线/氛围/机位）。
- 即：`定妆参考图(锁脸) + shot.content(当集服装+场景+动作)`。缺服装或缺场景的 shot 视为未写完。

## 4. 单角色出图——禁止一图两张正脸（flux 不支持多角色）
- **同一张图里，禁止出现 2 个角色的正脸。** flux/kontext 单参考图只能锁 1 张脸，两张正脸必崩。
- 需要两人同框时：**只让被喂参考图的主角色露正脸**，另一人用**背影 / 过肩 / 侧脸剪影 / 只出手/局部**，绝不给第二张正脸。
- 因此每场景**单人出图**为默认（flux $0.04）。原多角色 nano-pro 路线在本剧**停用**（除非用户另行指定）。

## 5. 真人质感·去塑料感
- 追求**真实照片质感**：自然肤质、细腻毛孔、真实光影、轻微胶片颗粒；**避免**塑料感/过度磨皮/CGI 感/油光蜡像（no plastic skin, no waxy CGI look, natural matte realistic skin）。
- flux 安全过滤易对真人黑图 → payload 必带 `safety_tolerance:"6"`（gen-image 已内置）。出图不满意**禁止擅自重试，先问用户**（成本铁律）。

## 6. 真实人景比例·透视（像真的用相机拍出来的）
- 人物在场景里的**大小、比例、透视、景深必须符合真实拍摄**：由**机位/焦距/人物与相机的距离**决定人物在画面中的占比，不是把人像"贴"在背景上。
- 明确写机位与景别：远景(全身+环境)/中景(腰以上)/近景(胸像)/特写(脸)；人与场景处在**同一透视与光照**下（同一光源方向、同一色温、地面/家具与人脚的接触真实）。
- 全身/远景时脚要落地、比例正确；两人同框时近大远小符合站位（配合第4条：仅主角露正脸）。
- 一句话：**先想"这台相机站在哪、拍多远"，再让人物以真实比例长在那个场景里**。

## 7. 说话人标签不传给模型（TTS 只读正文）
- 脚本里「旁白：」「灵兰：」「顾晏辞：」这类**说话人标签**只是元数据（对应 script.json 的 `voice` 字段），**绝不送进 TTS 合成、也不上屏**——只有冒号后的正文台词才朗读/显示。
- 即：`voice` 决定谁在说、用什么音色；`captions` 只放正文。标签写进正文=会被念出来「旁白冒号…」，属错误。

## 8. 语言与音色（本剧铁律）
- **旁白 = 越南语**（受众母语，引导叙事）。
- **人物说话（中国角色）= 中文**（同时是学习内容，字幕三行拼音/中文/越南语，逐字卡拉OK落在中文行）。
- **越南女主·灵兰的内心独白 = 越南语**（她灵魂是越南人，用母语思考）。
- 音色固定：旁白 `vi_female_ruan_uranus_bigtts`；灵兰独白 `vi_female_linh_uranus_bigtts`；中文角色音色以**豆包官方**为准，逐一免费试听验证后锁定（见 [[../../../public/library/audio/voices/README]]）。

## 9. 字幕：全程三行·全程卡拉OK双语（用户 2026-07-15 锁）
- **每一拍都是三行**：`拼音 / 中文(毛笔行楷 Ma Shan Zheng) / 越南语`。**没有只越南语的单行拍**——旁白、独白、对白**一律带中文行**。
- **★中文↔越南语一一对应（铁律）**：越南语拍的中文行=该句越南语的**完整对应翻译**（意思一一对应，不是短总结/概括）。**严禁"几个中文字对着一大段越南语"**。
- **★一行放不下就拆句顺序显示（铁律，用户 2026-07-15 定）**：一句在一行放不下（中文约 >10 字），就把它**拆成 2+ 个顺序拍**（同一 `sceneId` 共用同一张图），每拍一小句、短到一行放得下、中文↔越南语逐句对应，**这句播完接下句**。**不是换行、不是滚动条、不是缩小字号**——是分句次第出现。拆句在 `script.json` 里做（每小句一个 beat，voice/sceneId 不变，首拍带 shots）。
- **每一拍中文行都走逐字卡拉OK**（逐字弹起+点亮青色）：
  - 中文对白拍：用火山中文 TTS 的**真字级时间戳**驱动。
  - 越南语拍(旁白/独白)：中文行没有真时间戳 → `build.mjs` 用 `evenCharTimings()` **按本拍音频时长把汉字均匀铺开**驱动卡拉OK，扫字跟着越南语音频走。
- 越南语行**自动换行**（句长），不用单行缩放；中文行短句为宜（长了会被压小）。
- 字体：中文=`library/fonts/MaShanZheng.ttf`(毛笔行楷)；拼音/越南语=Nunito。版面照 chinese-learn(黑底/上图720横屏/下三行字幕)。
- 实现落点：`build.mjs`(按 vi_/zh_ 判语言合成 + evenCharTimings 补时间戳) + `src/layouts/chinese-drama.tsx`(三行渲染,中文行 KaraokeRow,越南语行换行块)。
