# 出图 / 出视频铁律（chuanyue-drama）

> 上位法 `/base/04-成本与铁律.md`、`/base/03-流水线与manifest.md`。本文是本模板私有落地。
> 出图与出视频**共用一套人物锚**：先出关键帧（nano-banana-pro/edit 喂定妆图保脸）→ 该帧作为 I2V 输入图。

## 1. 真人质感（画风）
- 一律 `prompts/style.md` 的电影级真人古装质感；禁动漫/塑料/3D/插画。女主绝色、肤白。

## 2. 一图一正脸（单张出图只锁 1 张正脸）
- 单张出图**只有 1 张正脸**。两人同框时，另一人只给**背影 / 过肩 / 侧脸 / 局部**，不出第二张正脸。
- 需要真·多人正脸同框：全片**至多 1 次** `nano-pro`（喂多张定妆图），其余单人反打解决（省钱铁律）。

## 3. 真实人景比例
- 人物大小 / 透视 / 景深符合机位与距离，像真拍出来，不是把人贴到背景上。

## 4. 服装与"露腿"场景化（★用户 2026-07-18 锁）
- 女主正式/朝堂/见长辈场合**端庄**（襦裙、披帛、发髻）。
- **只在合理场景露出又白又细的腿**：沐浴、寝衣薄纱、涉水、受伤裹伤验伤、习武短打、
  舞姬伪装、高开衩骑装等。露腿服从剧情，写进该拍 `{shot}`，不硬凹、不出戏、SFW 得体。

## 5. ★绝不写文字/边框/留白（扩散模型"提到什么画什么"）
- prompt 里**绝不**写 subtitle/text/caption/box/frame/watermark/logo/"留白给字幕"（含 `no box` 负面写法）。
- 三行字幕由 Remotion 后期合成在字幕区，画面只管画好内容。
- 不放身体部位负面词（`no extra fingers` 等会触发 fal nsfw 黑图）。

## 6. 出图路由（★全部 nano-banana-pro，用户 2026-07-18 锁，彻底放弃 flux）
| 用途 | fal id | ~$/张 | 参考图 |
|---|---|---|---|
| 首次出人物·定妆图 | `fal-ai/nano-banana-pro`（文生图） | ~0.15 | 无——从角色 canonical 文字生成 model-sheet |
| 角色变体(露腿版/现代版) | `fal-ai/nano-banana-pro/edit` | ~0.15 | ★喂该角色**非露腿定妆图**当参考，保同一张脸(人物一致) |
| 剧集关键帧·单角色 | `fal-ai/nano-banana-pro/edit` | ~0.15 | 喂该角色定妆图（保脸），场景/服装由 `{shot}` 现写 |
| 剧集关键帧·多角色同框 | `fal-ai/nano-banana-pro/edit` | ~0.15 | 喂多张定妆图；仍优先单人反打省画面 |
| 空镜/物证 | `fal-ai/nano-banana-pro/edit` | ~0.15 | 喂 styleAnchor 只借画风、绝不加人 |

- ★**新变体/新造型一律 edit + 该角色已锁定妆图作参考**，不用文生图另画脸（会不一致）。
- **出图比例统一 `image.aspectRatio="3:2"`**（banana 直接出 3:2 横图），放入 3:4 版面上媒体区；字幕在下区。
- flux 已彻底弃用（效果差/蜡感重，用户 2026-07-18 锁）；旧 flux 定妆图已删除。全部 banana-pro。

## 7. ★出视频（I2V / kling）铁律
- **流程**：① 先按 6 出该拍**关键帧**（含人物脸/服装/场景，已锁外观）→ ② 关键帧作为
  kling `image-to-video` 的**输入图**（`image_url`）→ ③ 视频 prompt **只写运动**（见 `video.tpl.md`）。
- **关闭原声（★用户 2026-07-18 锁）**：`generateAudio=false`，kling **不生成任何声音**；
  成片一律用我们自己的火山 TTS 配音，渲染层视频拍**静音播放**。
- **人物特征代入靠关键帧，不靠文字**：外观（脸、服装、发型）由关键帧决定，视频 prompt
  **不重描外观**，只描述动作/表情变化/镜头运动/环境动态。重描外观会让模型漂移换脸。
- 单段 **3–5s 整数秒**（默认 3.5s，落整数帧）；每集动态 ≥3 段、含开头 3s 钩子。
- **★尺寸铁律（用户 2026-07-18 锁）**：不管 kling 规则，最终交付固定 **3:2**。kling 只认 16:9/9:16/1:1，
  故 `video.aspectRatio=16:9` 喂 kling；`gen-video` **喂图前把 3:2 关键帧居中裁成 16:9**、
  **收片后把 16:9 成片两边裁掉回 `video.finalAspect=3:2`**（remotion 自带 ffmpeg，本地无成本、不算 fal）。
- 视频 prompt 同样**绝不写文字/字幕/边框**。

## 8. 花钱铁律（继承 /base/04）
- 每次 fal（出图/出视频）都花钱。结果有问题（黑图/漂移/换脸/画错/不满意）**禁止自动重试**，
  先报用户、经同意再调。`maxAttempts` 逻辑：单资产一次，不自动付费重试。
- 按 hash 缓存：改一句只重算受影响拍；出图/出视频命中缓存不重复烧钱。
