# HSKS 总纲领 · HSK 官方知识 → 越南受众高密度速览短视频

> 本文件是 `templates/hsks/` 家族的**唯一宪法**。任何子模板（`hsks-vocab` / `hsks-grammar` / `hsks-topic` / `hsks-task` / `hsks-hanzi`）都必须服从这里的机制与设计系统；子模板文档只写「它在本类知识上的落地」，不得推翻这里的铁律。
> 做本家族前先读 base 的 [[00-底层规律]] 与 [[01-创作准则]]。

---

## 一、使命：唯一衡量标准是「有用」

赢家 `hsk-ziyuan` 火，不是因为动画炫，是因为它对「想学中文的越南人」**有用 + 值得收藏**。但它只覆盖 HSK 知识的一个角落（单字字源）。

`hsks` 家族的使命：**把 HSK 官方全量知识，做成越南学习者会点赞、收藏、转发的高密度速览卡。** 内容有用是 1，动效美术是后面的 0——没有 1，再多 0 也没意义。

受众固定：**越南中文学习者**。所有释义/引导用越南语，所有知识用中文原文。

## 二、四条铁律（不可违背）

1. **零编造**：每一个知识点都**直接从 `/hsk/HSK*/*.csv` 抽取**。构建脚本只做「抽取 + 排版 + 配越南语释义 + 配图」，**绝不生成/杜撰知识**。语法例句必须用官方 `cases` 字段原文。
2. **出处徽标常驻**：每条知识点/每页都带来源角标，格式 `HSK{级}·{类}·{子类}`（如 `HSK3·语法·动作的态`、`HSK4·词汇·当代社会`）。权威感 = 可信 = 值得存。
3. **短 + 看不完**：单条视频尽量短（完播率），硬上限 **24 秒**，且**总时长必须是偶数秒**（用户铁律）。一屏高密度塞满一类知识，配合自动翻页 → 人眼读不完一页 → 划走前想再看 → **回看/循环 → 完播与时长双涨**。
4. **配音必有 TTS 且必须与画面严格同步**：每条视频核心内容必须朗读（中文 TTS，复用 `public/library/tts-*` 缓存库）。★**要么不发音，要么发音就必须字词与高亮严格对齐**——读到哪个知识点，哪个才高亮，绝不平均步进/胡乱跳动。BGM 可选，**不接受纯静音**。

## 三、节奏引擎（★所有子模板共用，写死）· 逐词等距节拍

**核心：高亮与发音严格对齐，且全程停顿完全均分。** 取代任何"固定秒数/平均步进/页尾补时"。

1. **分页只管版面密度**：子模板声明 `perPage`（每页知识点数）；`pages = clamp(ceil(itemsTotal / perPage), 3, 6)`。
2. **逐词配音 + 等距节拍**：每个知识点**单独配音**（可复用词库，如 `public/library/tts-vocab/`），全程每词一个**等长 slot** = `最长词音频 + gap`。于是**词间停顿 = 组(页)间停顿，完全均分**（不会出现组尾停顿比词间长）。
3. **高亮 = 逐词同拍**：每词在其 slot 起点播音并高亮，`[本 slot 起, 下一 slot 起)` 全程点亮（含均等停顿），读到哪亮到哪。
4. **总时长偶数秒**：`rawTotal = 词数 × slot` 向上取整到最近偶数秒，回算 slot 使节拍仍均匀；≤24s（超则减 `count` 或提 `speed`）。
5. **内容多少 → 用 count 控制**：知识点越多，视频越长（但仍偶数秒、≤24s）。子模板在 PLAN 里定"每条取多少"。

**翻页动效**：页间统一转场原语（见设计系统「翻页」），有明确「换页」感，强化「刚才那页没看完」的心理。

## 四、五个子模板（从 HSK 官方内容形态派生）

HSK 官方数据有 5 种真正独立的内容形态（`tree_nodes` 是 task/topic 的树状结构元数据，不单独成模板，只作 topic/task 的结构增强）。因此本家族 = **5 个子模板**：

| 子模板 | 知识源 CSV | 一句话主张 | 专属动效语言 | 配图 |
|---|---|---|---|---|
| **hsks-vocab** 词汇卡 | `01_vocabulary` | 一屏一批词，拼音+词+越南语+词性色标 | 卡片**瀑布/错峰弹入**，词性用颜色分区 | 每词 1 个彩色 SVG 卡通图标 |
| **hsks-grammar** 语法点 | `05_grammar`（含真实例句） | 语法结构公式化 + 官方例句 | 例句**打字机浮现**，关键词（了/把/被…）**脉冲高亮** | 场景 SVG |
| **hsks-topic** 话题图谱 | `04_topics`（三级树） | 一个主题域的知识网 | 中心词 → 分支**思维导图放射展开** | 主题场景 SVG |
| **hsks-task** 我能做到 | `03_tasks` | 「在 HSK{级} 你能…」交际能力清单 | 逐条**勾选打钩 ✓** + 进度感 | 情景小人 SVG |
| **hsks-hanzi** 部件识字 | `02_hanzi`（认读/手写） | 按部首/部件归类识字 | 部件**积木拼合**（区别于 ziyuan 的字源 morph） | 部件高亮 |

> 不与 `hsk-ziyuan` 冲突：ziyuan = 单字**字源演变**（象形→字）；`hsks-hanzi` = **部件/部首归类**的批量识字。两者角度不同。

## 五、生产顺序 + 系统化覆盖（★不可随意挑选）

**级别升序**：先把 **HSK1 全部知识**（跨全部适用子模板）做完，再 HSK2、HSK3……直到 HSK7-9。

**★覆盖账本（无重复、无遗漏、可审计）**：绝不手挑知识点。每个子模板维护一份**覆盖账本** `templates/hsks/<sub>/_coverage.json`，按 HSK 官方 CSV 的 `sort_order`（或 topic/task 树的固有顺序）**顺序推进游标**：
- 一条视频只声明「取下一批 N 个」（`script.json` 里 `{level, count}`），build **自动取该级别未覆盖的下 N 行**，标记已消费、推进游标、记录 `{videoId, from, to}`。
- **幂等**：重建同一 `videoId` 复用它已记录的区间，不再推进游标（避免重复消费）。
- 于是整个级别的知识点被**一条不落、一条不重**地切成连续批次；账本即进度表，随时可查"覆盖到哪了、还剩多少"。
- 越南语释义来自 `vi-lexicon.json`（词→越南语，可复用、可增量）；缺词时 build 报错列出，人工补全后再跑——**绝不让脚本臆造越南语**。

**★每个子模板必须有 `PLAN.md` 排期表**：明确「本模板一共出多少条、每一条精确取哪个范围的知识」。取数须完全确定（按 CSV `sort_order` 或树的固有顺序、固定每条份额），并给出各级视频数与命名规则（`hsks-<sub>-<level>-NN`）。无 PLAN 不得批量生产。

## 六、共享版式与设计系统

- **画布**：9:16 竖屏 1080×1920，fps 30。延续赢家竖屏形态。
- **左右安全区**：字幕/内容左右各留白 12%（≈130px），避开平台点赞/分享按钮（见 [[subtitle-safe-area-12pct]]）。
- **出处角标**：顶部或每卡角，`HSK{级}·{类}·{子类}`，常驻。
- **配色 tokens**（每个子模板可在此基调上取自己的强调色）：
  - 每级一个主题色系（HSK1 暖橙 → HSK6 沉紫，形成"进阶感"），具体色板在设计系统附录。
  - 词性/知识类别用**分类色**（动词/名词/形容词…各一色），保证一屏可辨。
- **字体**：中文用现有 `PingFang SC` / 已装中文字体；拼音+越南文用 `Nunito`（完整越南语声调）。沿用 ziyuan 的 `fonts` 约定。
- **配图**：**全部内联 React SVG 彩色卡通简笔画**，零成本、零 AI、跟数据走。风格：圆润、扁平、鲜明、简。图标库随子模板增量积累、跨视频复用。

## 七、分类动效设计系统（不同知识 → 不同动效）

统一的动效**原语**（primitives），子模板从中取用，形成各自的"动效语言"：

- **入场**：`cascade`（错峰瀑布，词汇）、`radiate`（放射，话题）、`typewriter`（打字机，语法例句）、`tick`（逐条打钩，任务）、`assemble`（部件拼合，汉字）。
- **强调**：`pulse`（关键词脉冲）、`glow`（辉光）、`colorPop`（分类色点亮）。
- **翻页**：统一转场（如整页向上抽走 + 下一页压入，或卡组翻面），全家族一致，强化"换页/没看完"感。
- **节拍**：TTS 读到哪条，哪条触发强调动效（读+看同步，视觉引导）。

所有动效必须服务「让知识不枯燥、一眼想收藏」，不得干扰阅读（12s 内信息要读得进去）。

**★炫酷是硬指标，不是可选项**：视觉必须"一眼想收藏"——入场之外要有持续的活力（扫光/高亮环/激活粒子/图标微动/背景流光/进度点脉冲）。同时**尽量为每个知识点配 SVG 彩色卡通简笔画**（`src/layouts/hsks-icons.tsx` 增量积累、跨视频复用），保证每张卡都有彩图、无空板。

**★图标铁律 · 绝不把汉字本身当图（通用，全家族遵守）**：SVG 必须是「这个词的**意义 / 用法**的抽象表达」，**绝不允许把和该词完全一样的中文字形画进画面**（即：屏幕上已经有汉字文字层，图标区再放一个一模一样的字＝错）。哪怕是"吧""边""不"这种看似画不出的虚词，也要想它的功能或使用场景来抽象。按词类给抽象原则（只给类型化概述，不写死具体某字怎么画）：
> - **具象实词**（多数名/动/形，茶·车·吃·大）→ 直接画那个物 / 那个动作场景。
> - **数 / 量词**（八·百·半·本）→ 用**数量意象**（阿拉伯数字 8/100、半圆、一摞书），不是写汉字。
> - **否定 / 程度 / 语气 虚词**（不·别·很·吧·吗·呢）→ 用**符号或语调场景**表功能：否定→禁止符🚫；语气助词→对话气泡＋跳动语调点；程度→强弱刻度条。
> - **方位 / 指示**（边·上·下·这·那）→ **高亮方位块＋指向箭头 / 位置关系图**。
> - **抽象关系词**（的·和·在·是）→ 用**连接线 / 归属 / 存在**的图示意象。
> - **交际短语**（不客气·谢谢·对不起）→ 画**使用该语时的场景 / 手势**（挥手、鞠躬、微笑、星光）。
> - **实在画不出** → 用「纯图形抽象占位」（彩色瓷砖＋抽象符号 / 光点 / 问号意象），**同样绝不写出该汉字**。fallback 里出现汉字＝违规。占位须**按词派生变体**（颜色/内形各不同），保证不同词的占位也不相同。

**★同一条视频里，任意两张 SVG 不得相同（用户 2026-07-23 · 禁止偷懒复用同一张图）**：两个不同知识点若渲染出同一张图＝错，必须给其一换不同抽象表达 / 专属图。落地：`src/layouts/hsks-icon-groups.mjs` 记「共用同一 case 的同义词组」；`_engine.mjs` 的 `assertDistinctIcons` 在 build 时检测重复——词/字/语法/话题卡（每项都是独立概念）**重复即报错**；task 技能图（听/说/读/写天然可复用）仅警告。抽象词 fallback 已按词变体、故不会互相撞。**新写共用 case 时务必同步 `hsks-icon-groups.mjs`。**

## 八、生产管线（复用现有多模板架构，不动通用脚本）

1. **子模板目录**：`templates/hsks/<sub>/{authoring.md, build.mjs, template.json}`。
2. **渲染层**：`src/layouts/hsks-<sub>.tsx` 导出 `LAYOUT`（见 `src/layouts/types.ts` 的 `LayoutModule`），在 `src/layouts/registry.ts` 注册。
3. **一条视频**：`catalog.json` 加条目（`template: "hsks-<sub>"`, `shard: 年/月/日`）→ 视频目录放 `script.json`（只写"抽哪一段 HSK 知识"的选择器，如 `{level:"HSK3", type:"语法", filter:"动作的态"}`）→ `node scripts/build.mjs <id>`（脚本自己去读 `/hsk` CSV 抽取、配 TTS、算分页、出 manifest）→ `npx remotion render <id> <out>`。
4. **零编造保证**：build.mjs 的数据来源**只有 `/hsk/*.csv`**；越南语释义走翻译库/人工，绝不让脚本"造"中文知识。
5. 改配色/节奏/字号/音色 → 改子模板 `template.json`，不动 `build.mjs` 逻辑、不动通用编排器。
