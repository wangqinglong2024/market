# 模板：chinese-learn · 中文学习字幕片（越南受众）

> **输入物是「一个无字幕中文视频」**（区别于其它模板的「文案」）。把原视频重排成竖屏学习版：
> 上方原视频、下方三行字幕（拼音 / 中文 / 越南语），逐词跳动高亮。
> 主用途：拿现成中文动画/影视（如葫芦娃）**混剪**成一条能在越南 TikTok 活下来的中文学习短片。

> 必然被划走。本文把剪辑重写成**由底层规律驱动的悬念设计**。做本模板前先读 base 的
> [[00-底层规律]] 与 [[01-创作准则]]，本文只是它们在"中文学习混剪"上的落地。

## 一、这条片子要赢在哪（先想清楚，再动剪刀）

我们不是"教一句中文"，我们是"设计一条越南人拇指停下、并且划不走、看到最后还学到中文"的时间线。

- **北极星**：3 秒留存 → 完播 → 评论。**先修钩子，再修结构，最后才谈教了几个词**（[[01-创作准则]] 七）。
- **触发器**：中文学习本身对越南学习者是「与我有关」（我在学中文）+「社交货币」（学会能显摆）。剪辑要再叠上**好奇缺口**和**高唤醒情绪**（故事的冲突/反转/萌点），凑够 ≥2 个触发器。
- **格式是资产**：本模板就是一个可复利的系列格式，每条是同格式的变体。别把某一条做成孤品，做的是"这个系列的下一集"。

## 二、剪辑设计准则（★ 核心，取代旧「随机剪辑/固定 80-20」）

### 1. 开头 3 秒：从**最抓人的一瞬**切入，不是从原片第 0 秒

- **禁止**按原片顺序老实地从头播。开场要么是全片**最高唤醒的画面**（冲突/惊变/萌点/名场面），要么是一句**制造好奇缺口的越南语解说**。
- 允许**冷开场（cold open）**：把后面一个最勾人的瞬间（哪怕原片里靠后）放在第一段，勾住之后再回叙。build 按 `clips` 数组顺序重建时间线，**数组第一段就是观众看到的第一秒**——把最狠的放这里。
- 第一屏字幕（拼音/中文/越南语）本身要能当钩子（平台靠 OCR/ASR 冷启，见 [[00-底层规律]] 三.1）：第一句就上最勾人的那句中文。

### 2. 层层悬疑：每一段都要有"不划走的理由"

- **开一个主缺口**：开头用画面或解说抛一个未解的问题（会发生什么？为什么？谁赢？），**答案压到结尾**。
- **每段之间续钩**：逐段自问「观众看完这一段，凭什么要看下一段？」——答不上来的段落就是流失点，要么删，要么用一句解说给它接一个钩（"但是……""结果没想到……""接下来这一下……"）。
- **绝不让画面变得可预测**：一旦观众觉得"后面我猜到了"，立刻划走。用剪辑制造反转预告、情绪爬升。
- **回环收尾**：最后一段/一句接回开头的缺口，诱发二刷。

### 3. 选段：完整的悬念弧，不是碎片拼盘

- 选出能自成 **起承转合** 的片段：有冲突、有升级、有揭晓。**判断、思考着选**，不是随便截。
- **compress，别 dump**：只保留推动悬念的镜头，把铺垫/重复/慢镜头剪掉。**时长能短就短**（默认目标 ≤60s，完播不行就再砍，见 [[01-创作准则]] 三）。
- 段与段之间靠**解说**桥接，保证压缩后故事仍连贯（见下）。

### 4. 原声 vs 解说：按**职责**分配，不是按固定比例

旧规则「80% 中文配音 + 20% 越南语解说」是拍脑袋的死配比，废除。改成**按职责决定谁在这一秒发声**：

- **原声（中文对白）是主体**：它是学习者要听要学的真实中文，也是画面的情绪现场。**能用原声承载的段落就用原声**，别用解说盖掉真人对白（盖了就没中文可学）。
- **越南语解说是"钩子 + 桥梁 + 悬念引擎"**，只在有明确职责时出现：
  1. **开场抛缺口**（第一段就位）；
  2. **桥接跳剪**——两段之间时间/情节跳了，用一句解说缝上，别让观众懵；
  3. **慢段前续钩**——原片要进入平淡处，提前用解说"预告下一个反转"把人留住；
  4. **点破难点**——出现越南人看不懂的文化梗时补一句。
- **解说落在无对白空档**（原片没人说话的 gap），别踩原声。
- **配比是"职责"的结果，不是目标**：自然结果通常是**原声占大头、解说是少量的接点**（大约原声主导、解说点缀）。若某条解说占比很高，多半是选段太碎、在用解说硬讲剧情——回去重选段，而不是加解说。

### 5. 解说写作：句句带钩，禁止平铺直叙

- ❌ 反例（旧 clips 里的写法）：「老爷爷讲起了灾祸的来历。」——这是说明书，不是钩子，观众划走。
- ✅ 正例：「爷爷突然压低声音——他要说的，才是真正的灾祸……」把**好奇缺口 / 悬念 / 情绪**塞进每一句解说。
- 解说是**越南语口语**、地道、有情绪，不要翻译腔；它的任务是"让人想看下一段"，不是"复述已经看见的画面"。
- 解说段也要给中文 `zh`（三行照常显示拼音/中文/越南语，供学习），但**音频读越南语**——所以 `zh` 也要写得像句好中文（学习价值），`vi` 写得像句好钩子。

## 三、版面（固定）

画布 **1080×1440（3:4）**，纯黑底 `#000000`（TikTok FYP 风）：

```
[黑底留白 120]
[原视频 1080×720  ← cover 裁切，focusY 调纵向焦点]
[间隙 120]
[字幕区 1080×360  ← 拼音行 / 中文行 / 越南语行]
[黑底留白 120]
```

## 四、字幕规则

- **每个标点切一张字幕卡**：一句话按 `，。！？；、` 切成多张卡，每张卡只显示一小句 → 短句才放得大、才好读（[[01-创作准则]] 三：文字是视觉重点）。
- **三行始终一致**：任何时候都是 拼音 / 中文 / 越南语 三行，不因段落类型变化。
- **逐词卡拉OK，三行一起动**：中文按 ASR 逐词时间戳跳动高亮；拼音跟中文同步；**越南语按本卡时长均匀卡拉OK**（越南语语序≠中文，无法逐字锁死，故均匀铺满本卡，视觉上和中文一起往前跑）。
- 三态配色：未读（dim 灰）→ 朗读中（karaoke 高亮 + 放大跳起）→ 已读（该行常色）。
- **字号取向（越南受众）**：越南语与拼音**偏大**（越南语是受众母语、拼音是发音抓手），中文**适中偏小**（认字为主、不必满屏）。具体值在 `template.json → captions.sizes`，渲染后按 :3000 观感微调。

## 五、特效规则（React 特效库是**开放**的，不固定）

`src/fx/` 是**开放、可扩展**的 React 特效库——想要新效果就写个 `useCurrentFrame()` 组件放进去、在 `EffectsLayer` 加一个 case 即接入（见 [[03-流水线与manifest]] 六）。**没有"只能用固定几个"的限制。**

- **layout 自带的节奏特效**（每条都有）：clip 转场「白闪 + 变焦冲击(zoom punch)」、字幕卡上滑淡入、每段 ken-burns 轻微推近。
- **按内容加的强调特效**：在情绪高点/反转/名场面那一拍，按需叠 `src/fx/` 的特效（或新写一个）放大冲击。**用哪个、用几个由内容定**，不设死上限；但别抢字幕这个主视觉（[[03-流水线与manifest]] 六）。
- **怎么知道该在哪一拍加、加什么**：靠**抽关键帧看画面**（见下节「看关键帧」）——先看清那一瞬画面里发生了什么（谁在喊、什么炸了、谁惊到了），再决定特效类型与位置，而不是拍脑袋。

## 五之一、看关键帧（★ 顺序铁律：先 ASR 懂文字，再"有目标"地截图）

**顺序不能反（用户 2026-07-13 锁定）**：

1. **先对全片跑 ASR** → `work/transcript.json`，**先把故事读懂**（谁说了什么、冲突在哪、名场面在哪、时间戳多少）。
2. **再"有目标"地抽帧**：只对 transcript 里**关键句 / 转折 / 名场面的时间点**截图，去确认那一瞬画面里到底发生了什么、够不够抓人。
3. 🚫 **禁止"先随机/按固定间隔盲截一批图"再看**——那是瞎截，既浪费又抓不到真正的戏。理解文字在先，截图是带着问题去验证画面。

用 Remotion 自带的 ffmpeg 从 `source` 原片抽指定时间点的截图：

```bash
# node_modules/.bin/remotion ffmpeg 就是打包好的 ffmpeg；抽单帧：
node_modules/.bin/remotion ffmpeg -y -ss <秒> -i "<原片.mp4>" -frames:v 1 -q:v 3 work/frames/t<秒>.png
# 例：抽第 293.5 秒那一帧
node_modules/.bin/remotion ffmpeg -y -ss 293.5 -i "public/videos/<shard>/<id>/葫芦兄弟.mp4" -frames:v 1 -q:v 3 public/videos/<shard>/<id>/work/frames/t293.png
```

- **对准 transcript 里的关键句时间点**抽帧（不是按固定间隔），确认名场面/冲突/萌点的画面——这是选段、排序、写解说、放特效的依据。
- 抽帧是**本地 ffmpeg、免费**，可随意多抽；截图放 `work/frames/`（不入成片）。
- 我（Claude）会先读 transcript、再读这些**有目标**的截图来规划这条片子，而不是盲剪。

## 六、clips.json 格式（混剪模式；视频目录放此文件即启用）

```jsonc
{
  "source": "path/to/原集.mp4",
  "clips": [
    // ★ 数组顺序 = 成片时间线顺序。第一段就是观众看到的第一秒——把最抓人的放这里。
    { "type": "zh", "startMs": 276900, "endMs": 281200,
      "viClauses": ["Ông ơi", "Cảm ơn ông", "..."] },   // 按中文标点分句，一一对应
    { "type": "narration", "gapStartMs": 293000,
      "zh": "爷爷突然压低了声音。", "vi": "Nhưng điều ông sắp nói mới thật đáng sợ..." }
  ]
}
```

- **`type:"zh"`**：原片段（`startMs~endMs` 取 ASR 整句区间，时间戳精确，保原声）。build 按标点切成多张卡，`viClauses` 与切出的中文小句一一对应。
- **`type:"narration"`**：原片空档（`gapStartMs` 起，时长=越南语 TTS 时长）。给 `zh`（中文句，供学习，写成好中文）+ `vi`（越南语句，写成钩子）；画面 3 行、音频读越南语、原声压低。
- **顺序即叙事**：`clips` 数组的顺序就是成片顺序，可**不按原片时间**（支持冷开场/重排叙事）；每段内部的字幕时间戳由 build 重映射到成片时间线。
- 前置：先对**全片**跑 ASR，结果放 `work/transcript.json`（`node scripts/asr.mjs 全片音频 > work/transcript.json`）。混剪 build 从这里取字幕时间戳。
- build 会 ffmpeg 逐段剪（快+准两级 seek）拼成 `source.mp4`，并把字幕时间戳重映射到混剪时间线。

## 七、整片模式（无 clips.json）

放 `source.mp4` + 可选 `vi.json`（每句越南语翻译），`node scripts/build.mjs <id>` 整片重排。
> ⚠️ 整片模式**没有剪辑设计**，只适合本身已经足够短、开头就够抓人的素材；一般题材优先用**混剪模式**按第二节做悬念设计。

## 八、制作步骤

1. 建 `public/videos/<shard>/<id>/`，放 `source.mp4`（整片）或 `clips.json` + 全片 `work/transcript.json`（混剪）。
2. **先跑 ASR 读懂故事 → 再对准关键句有目标地抽帧（§五之一 顺序铁律）→ 按第二节设计悬念弧**（选段 / 排序 / 写解说）→ 写 `clips.json`。这是最关键的一步，别跳，顺序别反。
3. `catalog.json` 加条目（template: `chinese-learn`）。
4. `node scripts/build.mjs <id>`。
5. Remotion Studio / 渲染预览，逐句听多音字读音（见 [[03-流水线与manifest]] 四），走一遍 [[01-创作准则]] 八自检。
6. **出封面（全局规则，见 [[03-流水线与manifest]] 七）**：从原片挑最抓人的一帧 → 写 `cover.json`（中文钩子大字 + 越南语钩子，按本条内容拟、要抓人）→ `remotion still src/index.ts cover <dir>/<id>-cover.png --props=<dir>/cover.json`。
7. **成片渲染**：`remotion render src/index.ts <id> <dir>/<id>.mp4`。★ 交付物命名规则（[[02-系统架构]] 五）：成片 `<id>.mp4`、封面 `<id>-cover.png`，直接命名、不要叫「成片/封面」。

## 九、语音成本记账

build 结束打印并写 `work/cost.json`：ASR 按全片音频时长×单价（默认 ¥1.5/小时估，精确以时长包单价为准）+ 越南语 TTS 条数。10 分钟原集 ASR ≈ **¥0.25**。

## 十、可调项（template.json）

- `source.region` / `source.focusY`：视频区大小与裁切焦点。
- `subtitle`：字幕区位置与高度。
- `captions`：三行颜色、`karaokeColor`、`dimColor`、字号 `sizes`、行距、解说 `duck` 原声音量、`narrationVolume` 解说音量。
