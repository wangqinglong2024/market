# 03 · Remotion 绘本页动效与排版

渲染层是**纯数据驱动**：一个万能 Composition 读 `manifest.json` 渲染，自身不含业务内容。
`calculateMetadata` 读 manifest 累加各页 `durationMs` 得总时长与 fps。

## 一段旁白内的多图过程动画
一个 page 可含 `shots[]`（1~N 张）。按 `weight` 把该页 `durationMs` 分给各 shot，
shot 间用交叉淡入/顺序揭示表现"过程"（如山底→山中→山顶的攀爬）。单图时退化为静态页 + Ken Burns。

**★ 反向：多拍共图（2026-07-07 新增，配合短句切拍）**：同 `sceneId` 的连续 beat 共用同一张图——
渲染层在这段区间**不换图、不转场、运镜不重启**（把共图区间当一个连续运镜时间轴），只有字幕/配音在逐句切换；场景切换处才转场。见 [[08-script-to-video-rules]] 第 1/2/6 步。

## "翻小人书"动感 = 这几样叠加（都不烧 API）
1. **翻页转场**：镜头间做翻书/卷页（CSS 3D `rotateY` 或卷曲）。"小人书"感的灵魂。
2. **Ken Burns**：单页内极缓慢推拉/平移，静图也有呼吸。
3. **Line boil（手绘抖动）**：每隔几帧让画面/线条极轻微位移+旋转，模仿逐帧手绘。便宜且出彩。
4. **纸张质感 + 暗角**：叠纸纹理 + vignette，强化绘本翻看感。
5. **视差（v2 可选）**：若 manifest 提供 `imageLayers{bg,character}`，角色与背景错位轻晃。成本翻倍，先不做。
6. **翻页音效 + 轻 BGM**：见 [[04-tts-captions]] 音频部分。

## ★★ 版式：3:2 横图合成到白底 3:4（2026-07-07 用户锁定，取代旧 1:1→9:16 十六等分）
画幅 **3:4 竖屏（1080×1440）**；出图**统一 3:2 横图**（`settings.image.aspectRatio="3:2"`）。
渲染器 `Video.tsx` 把横图放到**白底 3:4** 画布上，**上下对半分**：

```
┌────────────────────┐  0
│                    │
│    3:2 横版画面     │  1080×720 ← 满宽横图，AI 只管画这张横图
│                    │
├────────────────────┤  720
│                    │
│  字幕区（下半 3:2）  │  1080×720 ← 拼音/中文/越南语，1~N 行，整组垂直居中
│                    │
└────────────────────┘  1440
```
- **为什么改 3:4**：旧 9:16 全屏纯白底与 TikTok 等平台的**白色 UI 文字**重叠，页面发花；3:4 发布后平台文字落在视频外。
- **为什么图切 3:2**：整屏刚好上下各一半——上半放图、下半放字，版式简单确定。
- 字幕区**一次只显示一句短句**（单行拼音/中文/越南语，见下节），整组在下半区**垂直居中**。
- 出图提示词**只描述横版画面本身**（主体 + 大片纯白背景），**绝不提**框/字幕/留白（见 [[10-art-style-locked]] 4.4 死命令）。
- **★ 尺寸归一化照旧**：`build.mjs` 用 sharp 量白底上人物身高占比、算 `imgScale` 缩到统一目标，渲染层把横图 `scale` 乘 `imgScale`，所有拍人物一样大，四周露纯白画布。⚠️ 画布从方图变横图后，`settings.image.charTargetHeight`（旧 0.66 按方图标定）**需在第一条新视频上重新标定**。见 [[10-art-style-locked]] 4.6/4.8。
- 旧的「1:1 方图 → 9:16 十六等分」版式只属于已完成的 2 条旧视频，新视频一律用本节。

## 字幕排版（★ 2026-07-07 用户锁定：一屏一句短句、三层单行，文字是视觉重点）
字幕区一次**只显示当前正在读的这一句短句**，固定三层、**每层一行、绝不换行/堆多行**：
```
jiǎ zhōu jí zhě,          ← 拼音，声调符号 ā á ǎ à，单行
假舟楫者，                 ← 中文，主视觉，单行（逐字跳字）
Người mượn thuyền bè,     ← 越南语意译，单行
```
- 旧「一屏多行对（`captions.lines[]` 逐行中越对照）」机制**废除**（2026-07-03 规则只服务已完成的旧视频）：切句变细后（一拍≈4~12 字，见 [[08-script-to-video-rules]]），一屏永远只有一句，随配音逐句切换。
- 每句字幕入场可轻微淡入/上浮（快、别抢戏），停留窗 = 该拍时长。
- **★ 字体（2026-07-03 用户锁定）**：中文用 **SimHei 黑体**（粗、清晰，视频字幕更好认，取代旧 ZCOOL 快乐体）；拼音声调 / 越南语附加符号（ạ ầ ễ）用 **Nunito** 圆体覆盖。字体文件 `public/library/fonts/{SimHei.ttf,Nunito.ttf}`，`src/fonts.ts` 加载。
- **★ 字号（2026-07-07 改版，取代旧 67/42/50——单行大字，文字是视觉重点）**：中文 **92** / 拼音 **46** / 越南语 **54**。设计依据：左右安全边距各 60px（可用宽 960px），一拍 ≤10 个汉字时 92px 恰好从容放下；超出（11~14 字）由 FitLine 兜底整行缩，缩到底仍 ≥ 旧字号 67 的观感。首条新视频渲染后可微调，但**不再回到旧的小字号**。
- **★ 层间距与呼吸感（2026-07-07 设计）**：拼音与中文 gap **16**（注音紧贴其字），中文与越南语 gap **32**（分层清晰）；三层整组高 ≈240px，在 720px 下半区**垂直居中并整体上提 ~24px**（光学居中，底部略多留白更稳）。中文 letter-spacing 0.04em；中文行高 ~1.3 且跳字用 transform scale（不占布局），放大 1.2 不裁切、不推挤相邻字。
- **★ 拼音最小间距**：不同汉字的拼音不能视觉粘连。`RubyRow` 每个注音单元之间保留自然间距 `PINYIN_COLUMN_GAP = 12px`（随字号从旧 10 微调）；拼音行高 56 防声调符号被挤。若加间距后单行放不下，继续由 `FitLine` 量宽并整行等比缩小，优先保证不出框、不换行、不粘连。
- **单行不折行自适配**：每行 `white-space: nowrap`，`FitLine` 用 `useLayoutEffect` 量 `scrollWidth`、算 `scale=min(1,avail/measured)` 整行等比缩小贴合（配 `delayRender/continueRender` 保渲染确定性）。⚠️ 这是兜底——切句规则先保证短句本来就放得下（见 [[08-script-to-video-rules]] 第 1 步）。
- 停留时长 = 该拍 TTS 真实音频长度（音频驱动）。
- **★ 逐字跳字（卡拉OK，2026-07-07 用户锁定，时间戳已真机验证）**：读到哪个字，哪个字**放大跳一下**（spring 弹起+回落，同时点亮颜色）。时间源 = 火山 TTS 字级时间戳（`with_timestamp:1` → `addition.frontend.words[]`，规范见 [[04-tts-captions]]），build 写进 manifest 每拍 `charTimings[]`，渲染层按当前帧毫秒定位「正在读的字」。跳字只作用于**中文行**（拼音随其字对齐可同步点亮，越南语行不跳）。🚫 绝不逐字单独合成 TTS。

## ★ 运镜升级：组合运镜（已锁定，2026-07-02）
`config/motion.json` 预设支持组合字段：`scale`(推拉) + `panX/panY`(平移) + `driftX/driftY`(正弦漂移，让静图持续"呼吸") + `rotate`(轻微旋转) + `ease`(inOut 缓动)。
- 预设：`push-in / pull-back / pan-left / pan-right / climb-up / ken-burns / sway / pop / still`。
- 分镜按句意（`rules.byKeyword`）自动选 preset；渲染层 `Video.tsx` 按名播放，**改预设即调运镜，不动代码**。
- beat 间用 `@remotion/transitions`（淡入/滑入/翻页）做转场，不再只是 opacity 淡入淡出。

### ★ 多样化运镜铁律（2026-07-03 用户强调）
- **别全片一个运镜**：按每拍**实际场景/内容**选不同预设，让镜头有变化。
  - 朗读金句/郑重 → `push-in`；恍然大悟/情绪高点 → `pop`；讲道理铺陈 → `pan-left/right`；
    「越来越好/领悟登高」→ `climb-up`；平静空镜 → `ken-burns/sway`；收尾定格 → `still`。
- **相邻两拍尽量不用同一个预设**，避免观感呆板。`script.json` 逐拍显式写 `motion`（不确定再落到 `rules.byKeyword` 自动选）。
- 运镜是"没特效也好看"的主力（特效已精简到 ≤2/片，见 [[11-effects-spec]]），所以运镜的多样性更重要。
