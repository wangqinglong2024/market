# 00 · 总览：每日金句 → 中华经典文化短视频 自动化流水线

## 战略（2026-07-03 调整，取代旧「家庭情景剧」方向）
不再做家庭日常剧。改做 **中华经典文化宣传 · 每日金句**：
每条视频讲**一句**古文 / 经典名著 / 哲理句（用户每次只交一句，如「温故而知新，可以为师矣」），
由 AI 把它扩写成一条「孩子朗读 → 讲解意思 → 场景演示 → 文化收尾」的治愈短片。
**内容纯做文化普及**（讲清这句古文的意思与智慧）；软件推广靠账号/品牌沉淀，**视频里不打任何广告**。
主战场：越南 / 泰国 TikTok（宝妈 + 亲子受众），目标 **TikTok 爆款**。

## 输入 / 输出契约
- **输入**：用户只给**一句**古文/哲理句（中文）。可选 `videoId`、`lang`（默认 `vi`）。
- **输出**：`public/videos/<shard>/<id>/` 下的原始需求、转换脚本、配音、图、`manifest.json`、`成片.mp4`。
  逐层可追溯，见 [[12-video-folder-spec]]。

## 固定内容结构（每条金句片，25–40 秒；结构 2026-07-03 锁定。2026-07-07 起按**短句**切拍：拍数随句子数走、不设上限，**场景/图约 4~7 个**）
1. **孩子朗读金句**（开场钩子）：孩子**打扮成汉服**（**现代小孩穿汉服，不是古代人/古代场景**，见 [[10-art-style-locked]] 4.7），郑重念出这句古文（画面）。**配音永远是旁白（开朗姐姐），不是孩子在配音**——孩子只是画面里在"读"。
   - **两句/并列的原文按上句/下句逐拍朗读**（分开读、分开上字幕），但**整句诗可共用 1 张图**（2026-07-07 调整）。画面出哪个孩子/几张图**按具体文案场景设计**（旧「一男一女随机」规则已取消，2026-07-07）。**配音仍是统一旁白**，只是画面人物在变。见 [[08-script-to-video-rules]]。
2. **点破意思**（1 句）：用**大白话**解释这句话讲的是什么（不写「妈妈说」等引述词）。
3. **场景演示道理**（2–3 拍）：用具体画面演示这个道理（例「温故而知新」→ 一个人在灯下复习旧书，忽然眼睛一亮想通新东西）。
4. **文化收尾**：点出这句话的智慧/点睛（如点名成语、回味道理），温暖收束。

> **★ 纯文化普及，禁止任何营销/广告（2026-07-03 用户锁定）**：内容只做中华经典文化的传播与讲解，**不做**软件推广、下载 CTA、「一起学中文」等任何营销/广告话术与画面；也**不要**把道理硬拗到「学习/用工具」等推销角度。收尾回到文化本身。
> **配音统一用旁白「开朗姐姐」**（`voice=narrator`），全片一个声音，不做多角色分工（见 [[04-tts-captions]]）。角色区分只在画面上。

## 已锁定的核心约束
- **画幅（2026-07-07 用户改版，取代旧 9:16）**：竖屏 **3:4（1080×1440）**。原因：9:16 全屏纯白底发布后与 TikTok 等平台叠加的**白色 UI 文字**重叠，整个页面发花；3:4 让平台文字落在视频画面外。**已完成的 2 条旧视频不重做**，新规则只对新视频生效。
- **字幕（2026-07-07 用户锁定：一屏一句短句，文字是视觉重点）**：语音/字幕按**短句**拆细（一拍≈4~12 字），字幕区一次只显示当前这一句——**单行拼音 + 单行中文 + 单行越南语，禁止换行/多行堆叠**（旧 `lines[]` 多行对照废除）。图片按**场景**聚合，多个连续短句可共用一张图（如一句诗上下两句分开读、共配 1 图），见 [[08-script-to-video-rules]]。
  - 金句那句：中文原文 + 拼音 + **越南语意译**（不逐字直译，文言文直译到越南语生硬难懂，2026-07-03 用户确认）。
  - 讲解/叙述拍：现代中文 → 越南语口语化意译。
  - 语言**配置驱动**：只产中文，翻译/拼音自动补齐；加语言 = 加翻译目标，不动画面/代码。
- **配音**：永远中文，**全片统一旁白「开朗姐姐」**（不做多角色分工，2026-07-03 锁定）。配音语言不随字幕语言变。
- **画风**：治愈系手绘（粗铅笔线 + 暖蜡笔），**全片统一**，见 [[10-art-style-locked]]。金句朗读拍孩子(现代小孩)穿汉服。
- **出图纯白底 + 尺寸归一化（2026-07-03 锁定；比例 2026-07-07 改版）**：所有图**纯白底 `#ffffff`**（死命令）；出图统一 **3:2 横图**（取代旧 1:1 方图），渲染层合成到白底 3:4 画布——**上半 3:2 放图、下半 3:2 放字幕**；人物大小由渲染层用 sharp 量身高**归一化到统一尺寸**。见 [[10-art-style-locked]] 4.6/4.8、[[03-remotion-animation]]。
- **背景按场景设计、别搞傻**：背景贴合该拍内容、极简克制，不堆家具/不无脑户外/不画空盒子房间。见 [[10-art-style-locked]] 4.9。
- **人物出场铁律（2026-07-03 新增）**：**不是每个画面都要出主角**。只有**该句确实需要人物**时才出人物；讲道理的空镜（书本、灯、山、路…）就画物/景，不硬塞主角。
- **出图路由**：**无人物 / 单角色一律走 flux**（`fal-ai/flux-pro/kontext`）；仅**多角色同框**才用 nano-banana-pro。画风永远统一。见 [[05-cost-and-models]]、[[10-art-style-locked]]。
- **★★ 一条视频最多 1 次 banana（2026-07-05 用户锁定）**：全片**至多一个多角色拍**，即最多调 1 次 nano-banana-pro($0.15)，其余全 flux($0.04)；`build.mjs` 出图前校验 >1 报错。见 [[05-cost-and-models]]。
- **特效（2026-07-03 精简）**：全库只保留 4 个（ComicPops / EmojiRain / ScorePop / ZoomBlur）；**一条视频最多 2 个**，且**仅在内容需要时**才加，宁缺毋滥。见 [[11-effects-spec]]。
- **运镜**：**按场景做多样化运镜**（推拉/平移/攀升/摇摆…），别全片一个运镜，见 [[03-remotion-animation]]。
- **成本**：≤ $1 / 分钟视频（出图为主，语音成本忽略不计）。
- **Git 上传限制（2026-07-05 用户锁定）**：所有 `*.mp4` 一律在 `.gitignore` 中禁止提交上传；成片只在本地生成、预览和交付。
- **★★★ fal 出问题不许擅自重调（2026-07-05 用户锁定）**：任何 fal.ai 调用结果有问题，**禁止自己重新调用**，必须先告知用户、经用户同意才能再调。本地/免费步骤（TTS、本地合成 BGM、渲染、改 config）不受限。见 [[05-cost-and-models]]。
- **★ 背景音乐一律本地合成、禁止任何 API/fal 生成**（`scripts/gen-bgm.py`，见 [[04-tts-captions]]）。

## 固定角色（沿用锁定形象，见 [[10-art-style-locked]]）
哥哥(boy)、妹妹(girl)、爸爸(dad)、妈妈(mom)、狗狗(dog)。金句片主要用到 **孩子（朗读）+ 父母（讲解）**，其余按内容。

## 整体架构（两层分离）
**生成层（Node 脚本，可缓存可重跑）：**
```
一句古文
  → LLM 扩写 + 分镜（孩子朗读/父母讲解/场景演示/收尾）   script.json
      （每拍带：配音角色、是否出人物、场景、出图 prompt、三语字幕）
  → 翻译(越南语意译) + 拼音                              字幕数据
  → 火山 TTS 按角色配音                                  每拍 .mp3 + 真实时长
  → fal.ai 出图（无人物/单角色=flux；多角色=nano-banana-pro） 图片资产
  → 汇总                                                manifest.json
```
**渲染层（Remotion，纯数据驱动）：** `manifest.json → 万能 Composition → render → 成片.mp4`。
