# 00 · 总览：文案 → 中文学习宣传视频 自动化流水线

## 目标
输入一段中文文案脚本，自动产出一条「家庭日常短剧 / 绘本风」宣传视频，用于推广中文学习软件。
长期目标：批量出片，多语言版本，固定角色长期复用。

## 已锁定的核心约束
- **画幅**：竖屏 9:16（1080×1920）。主战场抖音 / 视频号 / TikTok / 越南本地短视频平台。
- **字幕**：每镜三行 —— 拼音 + 中文 + 当地语言（首发越南语，后续泰语等）。
  - 语言**配置驱动**：脚本只产中文，翻译/拼音自动补齐。加语言 = 加翻译目标，不动画面/代码。
- **配音**：永远中文，火山引擎「开朗姐姐」，旁白叙述方式。配音语言不随字幕语言变。
- **画风**：简笔画 / 水彩细线手绘漫画，多彩。**仅借鉴《小王子》的线条质感与水彩配色感觉，绝不使用书中任何角色形象或 IP 元素**（创意 + 法律）。
- **动效**：小动不要大动 —— 像翻阅小人书（翻页 + 轻微呼吸感），不做角色大幅动作。
  - 注意：一段旁白可对应**一张或多张**图，画面随文字内容推进（例「从山底爬到山顶」→ 山底/山中/山顶多图 + 爬的过程动画）。画面必须与该段文字内容相关。
- **角色**：固定角色长期复用 —— 两个胖娃（一男一女，**胖胖的、可爱、活泼、惹人喜欢**）、爸爸、妈妈、一只狗狗（宠物）。
- **成本**：≤ $1 / 分钟视频（出图为主，语音成本忽略不计）。

## 固定角色（待在 02 文档定稿 canonical 描述）

2. 妹妹：胖胖可爱活泼的女孩
3. 爸爸
4. 妈妈
5. 狗狗（家庭宠物，可爱）

## 整体架构（两层分离）
**生成层（Node 脚本，渲染之外，可缓存可重跑）：**
```
文案.txt
  → LLM 分镜（按场景聚合，不逐句）      script.json
  → 翻译(越南语) + 拼音                  字幕数据
  → 火山 TTS 合成中文                    每段 .mp3 + 真实时长
  → fal.ai 出图（有主角色=参考条件生成 / 其它或无人物=文生图）  图片资产（每段 1~N 张）
  → 汇总                                manifest.json
```
**渲染层（Remotion，纯数据驱动）：**
```
manifest.json → 万能 Composition（calculateMetadata 读时长）→ render → 成片.mp4
```
原则：调 API / 产资产全在生成层跑完；Remotion 只「拿现成数据画画」，不含业务内容。
