# 06 · 待验证的高风险未知（先验证再开发）

按风险高低排序。最高风险应在写流水线代码**之前**用真实调用 spike 掉。

## ✅ 1. 角色 + 画风一致性（已验证通过 2026-06-28）
- 假设：nano-banana 参考图条件生成能稳定保住固定角色 + 画风。**结论：成立。**
- Spike 结果：生成全家福定妆图（`public/library/characters/_spike-lineup.png`）→ 用其作参考，
  nano-banana/edit 把两个孩子放进全新厨房早餐场景（`public/videos/_spike-kitchen.png`），
  脸/发型/体型/衣服/画风均保持一致。
- 退路（暂不需要）：flux-kontext / 训练角色 LoRA（见 [[02-character-consistency]] F）。

## 🟠 1b. 出图比例（部分解决）
- ✅ 文生图 nano-banana 支持 `aspect_ratio: "9:16"`，已验证输出 768×1344。
- ⬜ 待验：参考条件生成 nano-banana/edit 是否也接受 `aspect_ratio`（之前不传时沿用了参考图的 1:1）。
  退路：角色圣经直接做成 9:16 参考 / 9:16 画布合成。

## 🟠 2. 火山 TTS 字级时间戳
- 是否返回字/词级时间戳？决定能否做逐字高亮跟读（[[04-tts-captions]]）。
- 无则降级为整句淡入淡出。

## 🟠 3. 出图比例与 9:16 对齐
- 模型出图比例能否直接出竖屏，或需指定尺寸/裁切策略，且构图能给底部字幕留白。

## 🟡 4. 多音字拼音准确度
- 选定拼音库后抽样验证；必要时 LLM 按词义校正。

## 🟡 5. 越南语翻译质量 + 字体覆盖
- 译文是否自然口语；选定字体是否完整覆盖越南语 + 拼音声调（避免豆腐块）。

## 🟡 6. review gate 形态
- 全自动 vs 半自动人工过图。初期建议半自动，保证出片质量。

## 建议的动手顺序
1. 跑 **#1 一致性 spike** ← 下一步
2. spike 通过 → 定稿角色圣经
3. 搭生成流水线（分镜/翻译拼音/TTS/出图/manifest）
4. 搭数据驱动 Remotion 模板
5. 端到端跑通 1 分钟样片
